**5. Project Structure:**
```
src/
├── components/      # Reusable components (Header, Footer, CaseStudyCard, etc.)
├── content/         # Markdown content collections
│   ├── case-studies/    # Portfolio case studies with images
│   └── writing/         # Blog posts and articles

├── pages/           # Route pages (index, about, work, writing, contact, 404)
├── scripts/         # Client-side scripts (imageViewer.ts)
└── styles/          # Global CSS (design system tokens, typography)

public/              # Static assets (favicon, web manifest)
.github/workflows/   # GitHub Actions for deployment
astro.config.mjs     # Astro configuration
```

**6. Content Management:**

**Adding case studies:**
1. Create markdown file: `src/content/case-studies/project-name.md`
2. Add frontmatter (see schema below)
3. Create image directory: `src/content/case-studies/project-name/images/`
4. Export images per IMAGE-GUIDELINES.md specs
5. Reference images: `![alt](./project-name/images/image.png)`

**Case study schema:**
```yaml
title: "Project Title"
description: "Brief description"
role: "Your role"
timeline: "Duration"
tags: ["Design Systems", "AI Tools"]  # Use Title Case
confidentiality: "public"  # or "partial" or "nda"
coverImage: ./project-name/images/hero.png  # 1600×900 (16:9)
```

**Adding writing:**
Same pattern as case studies, but in `src/content/writing/`

**Image guidelines:**
- Hero images: 1600×900 (16:9) for detail page tops
- Card images: Source as 1600×900, displays cropped to 4:3 on cards
- Inline images: Variable aspect ratio, max 1200px wide
- See IMAGE-GUIDELINES.md for complete specs

**7. Deployment:**

**GitHub Pages (automatic):**
- Push to `main` branch triggers GitHub Actions workflow
- Workflow: `.github/workflows/deploy.yml` (withastro/action@v3)
- Deploys to: https://chaddunbar.com
- SSL certificate: Auto-provisioned by GitHub (HTTPS enforced)

**Custom domain (GoDaddy):**
- DNS A records point to GitHub Pages IPs
- CNAME file in `public/` directory
- Certificate provisioned automatically (10-30 minutes)

**Manual deployment:**
```bash
npm run build
# Output in dist/ directory
```

**8. Design System:**

**Color tokens:**
- Adaptive light/dark mode (respects `prefers-color-scheme`)
- Warm neutral palette with slate blue accent
- CSS custom properties in `src/styles/global.css`

**Typography:**
- System font stack: Inter, -apple-system, SF Pro, Segoe UI
- Fluid type scale (clamp for responsive sizing)
- Editorial hierarchy with generous whitespace

**Components:**
