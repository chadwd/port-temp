## Contributing

When learning something useful about Claude Code:
1. Create a new markdown file with clear, practical guidance
2. Include decision frameworks (when to use what)
3. Add real examples from this project
4. Note anti-patterns to avoid
5. Update this README with the new document

## Format Guidelines

Each knowledge document should include:
- **Title and purpose** - What problem it solves
- **Core concepts** - The key things to understand
- **Decision framework** - How to choose between options
- **Examples** - Real, working examples from this project
- **Anti-patterns** - What to avoid and why
- **Quick reference** - Condensed version for fast lookup
